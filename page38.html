Rules as Code

Project Design

  1. Concept Model - First, the team held a series of workshops to identify key concepts in the regulation and define the relationships between them.
  2. Decision Tree - Next, we converted the regulations into a decision tree to determine the effect of certain variables on vacation pay entitlement. 
  3. Coding and Prototype Development - After completing the models, our private sector partner encoded the regulations and developed a simple prototype based on the coded rules.

Future Learning 

The demonstration project was effective. It quickly and cheaply demonstrated that it is possible to convert legal text into machine-usable code. Building on this success, there is more to learn as we explore how rules as code could be applied .

What strategies or techniques could be used to accommodate more subjective or ambiguous rulesets in a rules as code process?

What would it look like to demonstrate an entire use case (including development of an API and a public-facing end product)?

What approaches, vocabularies, languages, and platforms should be standardized as we move forward?

What capacity and skillsets would be required for drafters and for regulators to be able to effectively work alongside someone with technical coding expertise?

How would a rules as code process change the process of drafting a new ruleset if done in parallel?
