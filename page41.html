Rules as Code

Current Rules as Code Demonstration Project: Motor Vehicles Hours of Work Regulations

To jump forward off of our first Rules as Code discovery project and prototype, we are now working with a multi-disciplinary team from across the GoC headed by International Rules as Code Subject Matter Expert Pia Andrews on a new project. Here we plan on taking what we learned the first time a step further and will apply the full Rules as Code process to the new Motor Vehicles Hours of Work Regulations.

We are set to begin a 3-week project sprint in February 2021 where we will complete concept models, decision trees, pseudo code, and source code for the new regulation. This will result in a complete human and machine readable version of a new regulation which we plan on testing through policy scenarios to gauge if the regulation itself meets the original policy intent.

What’s New With This Rules as Code Project 

• First time any Government will be applying the full Rules as Code process to a new regulation

• Incorporating story-telling and collecting lessons learned from day one of project life to release products along the way and more formally to the broader GC upon project completion

• First time any government will be utilizing virtual and open source collaboration tools to implement the full Rules as Code process

• Packaging tools, resources, templates, drafts, and meeting materials as a formal “Rules as Code Playbook: A How-to-Guide for the Canadian Context” for release upon project completion
 


Digital Transformation: What Can Rules as Code  Accomplish?

• Coded rules could be used to run “simulations” or put through a “policy engine” to test rules before they are enacted. For example, run demographic profiles of 200 Canadians through a coded ruleset for a benefits program to see who is eligible and whether your policy intent is met

• Human and Machine Readable versions of regulations support greater compliance, improved service delivery, lower compliance costs, less confusion/mis-interpretation of rules


• Most importantly the involvement of developers and others in the rule making process will lead to better drafted rules designed for digital systems from the start rather than as an after thought. Making the rule making process more transparent accessible and democratic
